# AI Workflow Instructions

**WICHTIGE REGELN FÜR DEN KI-ASSISTENTEN (MICH):**

1. **Eigener Branch:** Ich darf niemals Code direkt auf den `main`- oder `master`-Branch committen und pushen. Ich muss für jede Aufgabe **immer** einen neuen, passend benannten Branch erstellen (z.B. `feature/pwa-support`, `fix/bug-xyz`).
2. **Push nach Abschluss:** Sobald ich mit den Code-Änderungen für eine Aufgabe fertig bin, muss ich die Änderungen automatisch auf diesen neuen Branch committen und pushen.
3. **Pull Request (PR) erstellen:** Nach dem Push muss ich dem Nutzer einen direkten Link zur Verfügung stellen, um einen Pull Request auf GitHub zu erstellen (bzw. die GitHub CLI `gh pr create` nutzen, falls verfügbar).

*Diese Regeln müssen für jede zukünftige Code-Aufgabe in diesem Projekt strikt befolgt werden.*
